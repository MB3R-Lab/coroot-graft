# coroot-graft v1.0.0

`v1.0.0` is the first stable coroot-graft release. It keeps architectural
resilience analysis stable while making current service outages visible in the
managed Coroot dashboard.

## Highlights

- Separate stable topology and runtime activity windows.
- Services remain in Bering and Sheaft artifacts across short outages.
- Endpoints whose blocking dependency path contains an unobserved service are
  forced to availability `0` in the effective coroot-graft report.
- Effective gate, profile, aggregate, and endpoint metrics now reflect runtime
  path impact.
- New dashboard panels show recently observed services and endpoint runtime
  availability.
- Raw Sheaft artifacts remain available separately from runtime-adjusted
  coroot-graft artifacts.
- Sheaft is updated to `v1.1.0`, adding `fixed_k_replica_slots`; Bering remains
  pinned to `v1.0.0` on the stable `1.3.0` model/snapshot contract.

## Runtime behavior

The shipped configuration uses:

- `time_window: 1h` for stable topology
- `activity_window: 2m` for current service observation
- `interval: 1m` for the default Helm and example project

After a service leaves the activity window, the next sync marks affected
blocking endpoint paths unavailable. Async dependencies do not block endpoint
runtime availability.

New metrics:

- `coroot_graft_service_observed`
- `coroot_graft_endpoint_runtime_available`

New project artifact routes:

- `/api/v1/projects/{project}/report` — effective report
- `/api/v1/projects/{project}/sheaft-report` — raw Sheaft report
- `/api/v1/projects/{project}/summary` — effective summary
- `/api/v1/projects/{project}/sheaft-summary` — raw Sheaft summary
- `/api/v1/projects/{project}/activity` — runtime observation and impact

## Configuration

`coroot.activity_window` is optional and defaults to `2m`. A project can
override it with `projects[].activity_window`. The activity window must not be
longer than the effective topology `time_window`.

## Compatibility baseline

- Coroot server `1.18.8`
- Coroot cluster agent `1.6.1`
- Bering `v1.0.0`
- Sheaft `v1.1.0`
- Bering model/snapshot contract `1.3.0`

Exact source refs and immutable image digests are published in
`compatibility-manifest.json` and `toolchain.env`.
